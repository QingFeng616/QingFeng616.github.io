## 修复主页卡片链接（HTML `<a>` 不会自动转 hash 路由）

- **根因**：用户说侧边栏/导航栏点击地址正确，但首页(README.md)卡片点击不正确
- 差异在于：侧边栏是 **markdown 语法链接** `[x](notes/linux.md)`，docsify 渲染时自动转成 `#/notes/linux`（hash 路由），正常；首页卡片是**原生 HTML `<a class="card" href="notes/linux">`**，docsify 不会对 HTML `<a>` 的 href 自动加 `#/`，点击触发浏览器整页跳转到 `qingfeng616.github.io/notes/linux`，GitHub Pages 返回 raw 文件或 404
- **修复**：把所有卡片链接改成显式 hash 格式 `href="#/notes/linux"`（含 汇编/通信协议 等中文路径、about/contact/Diary 等）
- 顺带把 `_sidebar.md`/`_navbar.md`/子目录侧边栏/`_coverpage.md` 里的 `.md` 后缀也去掉（markdown 链接更规范，无害）
- **docsify 链接规则（关键坑）**：markdown 链接 `[x](notes/linux)` 或 `[x](notes/linux.md)` 都会被渲染成 `#/notes/linux`；但**原生 HTML `<a href="notes/linux">` 不会**，必须写 `<a href="#/notes/linux">`
- 修改落在 main 分支工作区（未 commit）；线上若从 master 部署需把修改同步到 master 再 push
