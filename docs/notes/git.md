# git 学习

## 基础命令
### 1. 配置与初始化

在使用 Git 前，通常需要配置用户信息并创建或获取仓库。

- `git config --global user.name "你的名字"`：配置全局用户名（提交记录中显示）。
- `git config --global user.email "你的邮箱"`：配置全局邮箱。
- `git init`：在当前目录初始化一个新的本地 Git 仓库。
- `git clone <远程仓库URL>`：克隆一个现有的远程仓库到本地。

### 2. 代码提交（基本工作流）

这是日常开发中最频繁使用的操作，遵循"暂存 -> 提交"的流程。

- `git status`：查看当前工作区和暂存区的文件状态（新增、修改或删除）。
- `git add <文件名>` 或 `git add .`：将指定文件或所有变动添加到暂存区。
- `git commit -m "提交信息"`：将暂存区的内容正式提交到本地仓库。
- `git diff`：查看工作区与暂存区之间具体的变更内容。

### 3. 分支管理

Git 的分支功能非常轻量且强大，常用于并行开发。

- `git branch`：列出所有本地分支（带 `*` 号的为当前所在分支）。
- `git branch <新分支名>`：创建一个新分支。
- `git checkout <分支名>` 或 `git switch <分支名>`：切换到指定分支。
- `git checkout -b <新分支名>`：创建新分支并直接切换到该分支。
- `git merge <分支名>`：将指定分支合并到当前分支。
- `git branch -d <分支名>`：删除本地分支（需确保已合并）。

### 4. 远程仓库协作

用于与 GitHub、GitLab 等远程平台进行代码同步。

- `git remote -v`：查看当前关联的远程仓库地址。
- `git remote add origin <仓库URL>`：关联一个远程仓库（通常命名为 origin）。
- `git push origin <分支名>`：将本地分支的提交推送到远程仓库。
- `git fetch origin`：从远程仓库下载最新的代码更新（不自动合并）。
- `git pull origin <分支名>`：拉取远程代码并自动合并到当前分支（等价于 fetch + merge）。

### 5. 撤销与回退

当操作失误时，可以使用以下命令进行补救。

- `git checkout -- <文件名>`：撤销工作区中未暂存的修改（恢复到最近一次 add 或 commit 的状态）。
- `git reset HEAD <文件名>`：撤销暂存区的修改（即撤销 git add 的作用）。
- `git reset --hard HEAD`：撤销工作目录中所有未提交文件的修改内容。
- `git log`：查看提交历史记录，以便找到需要回退的版本号。
